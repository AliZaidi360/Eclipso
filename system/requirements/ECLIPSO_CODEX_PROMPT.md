# ECLIPSO — Full-Stack Pre-Launch System
## Complete Build Prompt for Codex / AI Coding Agent

---

## OVERVIEW

Build a complete pre-launch web system for a product called **ECLIPSO** — an EDC (Everyday Carry)
multitool launching on Kickstarter. The system has three parts:

1. A **landing page** to build hype and capture email leads
2. A **pre-order / payment page** for reservation deposits
3. A **Node.js backend** that stores emails and orders in Google Sheets and processes payments via Stripe

---

## TECH STACK

- **Frontend:** Vanilla HTML + CSS + JavaScript (no frameworks, single files)
- **Backend:** Node.js + Express
- **Payments:** Stripe (PaymentIntents API + Webhooks)
- **Storage:** Google Sheets API v4 (via googleapis npm package)
- **Deployment target:** Backend on Railway or Render, frontend on Netlify

  appearance: {

    variables: {
      colorPrimary: '#e8c44a',
      colorBackground: '#1e1e23',
      colorText: '#e8e8f0',
      fontFamily: '"DM Mono", monospace',
      borderRadius: '0px'
    }

// Payment flow:
// 1. User clicks submit
// 2. Validate email + name client-side
// 3. POST to /create-payment-intent with { version, email }
// 4. Get clientSecret back
// 5. stripe.confirmCardPayment(clientSecret, { payment_method: { card, billing_details } })

// 7. On error → show error message, re-enable form

## BACKEND (eclipso-backend/server.js)

### Dependencies:
```json
{
  "express": "^4.18.2",
  "cors": "^2.8.5",
  "helmet": "^7.1.0",
  "stripe": "^14.14.0",
  "googleapis": "^140.0.0",
  "dotenv": "^16.3.1"
}
```

### Google Sheets Setup:
- Use `googleapis` npm package
- Auth via Google Service Account (client_email + private_key from env)
- Two sheet tabs: `Leads` and `Orders`
- Helper functions: `appendRow(tab, row)`, `getRows(tab)`, `emailExists(tab, email)`

### Leads tab columns:
```
Email | Source | Timestamp ISO | Timestamp PT (America/Los_Angeles)
```

### Orders tab columns:
```
Email | Version | Amount ($XX.XX) | Currency | Stripe Payment Intent ID | Status | Timestamp ISO | Timestamp PT
```

### Endpoints:

**POST /signup**
```
Body: { email: string, source?: string }
- Validate email format
- Check if email already exists in Leads tab (don't duplicate)
- Append row to Leads tab
- Return: { success: true, message: "You're on the list." }
- On duplicate: return success anyway (don't alarm user)
```

**POST /create-payment-intent**
```
Body: { version: 'aluminum' | 'titanium', email: string }
Prices:
  aluminum: $49.00 (4900 cents)
  titanium: $79.00 (7900 cents)
- Validate version and email
- Create Stripe PaymentIntent with amount, currency: 'usd', receipt_email, metadata: { version, email, product: 'eclipso_preorder' }
- Return: { clientSecret: intent.client_secret }
```

**POST /webhook** (raw body — must be before express.json())
```
- Verify Stripe signature using STRIPE_WEBHOOK_SECRET
- On event 'payment_intent.succeeded':
  - Extract email, version from metadata
  - Append row to Orders tab
  - If email not in Leads tab, append to Leads tab with source: 'preorder'
  - Log: "✓ Order saved: email — version — amount"
- Return: { received: true }
```

**GET /admin/leads** (requires x-admin-secret header)
```
- Read all rows from Leads tab
- Return: { count, leads: [{ email, source, signed_up }] }
```

**GET /admin/orders** (requires x-admin-secret header)
```
- Read all rows from Orders tab
- Calculate total revenue (sum of Amount column)
- Return: { count, total_usd, orders: [{ email, version, amount, stripe_id, status, created }] }
```

**GET /health**
```
Return: { status: 'ok', ts: new Date() }
```

### Middleware order (critical):
1. helmet()
2. cors({ origin: process.env.ALLOWED_ORIGIN })
3. POST /webhook with express.raw() ← must be BEFORE express.json()
4. express.json()

### Environment variables (.env.example):
```env
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
GOOGLE_SHEET_ID=your-sheet-id
GOOGLE_CLIENT_EMAIL=service-account@project.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----\n...\n-----END RSA PRIVATE KEY-----\n"
PORT=3001
ALLOWED_ORIGIN=https://yourdomain.com
ADMIN_SECRET=random-secret-string
```

## IMPORTANT IMPLEMENTATION NOTES

- **No frameworks** — pure HTML/CSS/JS for frontend files
- **No build step required** — open HTML files directly in browser for dev
- **Single file per page** — all CSS and JS inline in each HTML file
- **Stripe webhook raw body** — express.raw() must be registered BEFORE express.json() or webhook signature verification will fail
- **Private key newlines** — replace \\n with actual newlines when constructing GoogleAuth: `process.env.GOOGLE_PRIVATE_KEY?.replace(/\\n/g, '\n')`
- **CORS** — backend must allow requests from your frontend origin
- **Admin security** — protect /admin/* routes with x-admin-secret header check
- **No border-radius** — the brand uses sharp corners everywhere
- **cursor: crosshair** — apply to body in both pages
- **Google Fonts** — load Bebas Neue, DM Mono, Barlow Condensed from fonts.googleapis.com
