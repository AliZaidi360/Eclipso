# GEO Project Status

This repository contains the pre-launch web system for the GEO multitool product. Below is a summary of work completed so far and the remaining tasks for the team.

## Completed Work

- **Landing Page (`geo-landing.html`)**
  - Fully styled with dark tactical design and branded typography.
  - Email capture form implemented with offline queue fallback.
  - Animations, scroll effects, and responsive layout done.

- **Payment Page (`geo-payment.html`)**
  - Version selector (V1 aluminum, V2 titanium) with pricing.
  - Stripe Card Element integration and client-side form handling.

- **Backend (`geo-backend/`)**
  - Node.js + Express server scaffolded.
  - API implemented:
    - `POST /signup` for lead capture
    - `POST /create-payment-intent` to create Stripe PaymentIntents
    - `POST /webhook` to handle Stripe webhook events
    - `GET /admin/leads` and `/admin/orders` protected by `x-admin-secret`
    - `GET /health` health check
  - Google Sheets integration set up for leads and orders.
  - Price map synced between backend and frontend.
  - Environment variables documented in `.env` (not committed).

- **Design System & Guidelines**
  - Color palette, typography, cursor behavior, and layout conventions documented in `.github/copilot-instructions.md`.
  - Animation and form-handling patterns specified there as well.

## Notes

- The project uses plain HTML, CSS, and vanilla JavaScript—no build tools or frameworks.
- All styling is inline within HTML files for portability.
- Form handling uses a localStorage queue (`queueKey = "qtool_leads_queue"`).
- Backend environment variables:
  - `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `GOOGLE_SHEET_ID`,
  - `GOOGLE_CLIENT_EMAIL`, `GOOGLE_PRIVATE_KEY`, `ALLOWED_ORIGIN`,
  - `ADMIN_SECRET`, `PORT`.
